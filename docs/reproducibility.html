Sara Keeble (13:23:13): > Hi@Charlie George, I actually wasn’t the one who chose airflow for our team, so I can’t offer much in the way of comparisons. However, I’ve been happy with the performance relative to Luigi, which is what we were using before airflow. The main pipeline we’ve developed in airflow is complex and has to be run frequently, so the most important features for us are the ability to 1) handle tasks with many dependency relationships, and 2) little hands-on management time. The previous iteration of the pipeline in Luigi required a lot of babysitting and manual restarting. Of course, some of the burden of managing babysitting time falls on writing good idempotent tasks. The final thing that’s important for us is integration with Jenkins, which is what we use for automated regression testing. Overall, I think it works really well for production-level code.
